## 结论

当前模型确实存在明显校准问题，尤其集中在比分矩阵和总进球层。

本次检查不修改核心模型参数，不开放正式 EV，不开放模拟舱买入。当前结论是：模型可继续作为研究审计工具，但不能作为正式模拟下注模型。

初步判断：`pbase` 对平局偏低，对客胜略偏高。样本很少，不能直接拟合最终参数，但方向值得重点跟踪。

## 分赛事观察

样本不足，仅作方向判断：

- 国际友谊赛：10 场，平均预期总进球 3.55，实际 2.40，平均误差 -1.15
- 法国足球甲级联赛标签样本：5 场，平均预期总进球 2.85，实际 3.00，平均误差 +0.15
- 英格兰足球超级联赛标签样本：2 场，平均预期总进球 4.23，实际 1.50，平均误差 -2.73

同时发现赛事中文展示存在污染：例如 Armenia 的 `Premier League` 被展示成“英格兰足球超级联赛”，Congo-DR 的 `Ligue 1` 被展示成“法国足球甲级联赛”。这主要影响分联赛复盘和报告展示，不直接影响当前核心公式，但必须修复。

## 主要问题定位

1. 正式校准样本不足。
   目前只有 15 场合格赛前样本，远低于 100 场门槛。

2. `pshr` 还不能作为正式概率。
   校准区只有 3 场，拟合出的市场权重为 0.0，说明当前 `pshr` 实质仍等于 pbase。

3. 比分矩阵总进球偏高。
   这是大小球 EV 偏差的主要来源。

4. 平局概率偏低。
   这会影响胜平负概率质量，也会间接影响比分矩阵低比分区间。

5. 联赛名称本地化有污染。
   同名赛事需要按国家区分，不能把所有 `Premier League / Ligue 1 / Serie A` 都映射成欧洲五大联赛。

## 建议修正路径

### 阶段 1：先修数据展示和校准入口

- 修复国家相关的赛事名称映射。
- 在模型审计页明确显示：胜平负校准样本不足；大小球/让球未完成比分分布校准。
- 将大小球 / 让球研究 EV 标注为“比分矩阵未校准，不可执行”。

### 阶段 2：建立总进球校准候选

先不改正式模型，新增候选审计：

```text
mu_total_raw = mu_home + mu_away
mu_total_calibrated = a_league * mu_total_raw + b_league
```

在样本不足时，可先使用全局保守收缩：

```text
mu_total_candidate = 0.80 * mu_total_raw
```

但该候选只能进入审计，不得进入正式 EV。

### 阶段 3：重分配主客进球

总进球校准后，保留原始主客强弱比例：

```text
share_home = mu_home / (mu_home + mu_away)
mu_home_candidate = mu_total_candidate * share_home
mu_away_candidate = mu_total_candidate * (1 - share_home)
```

再重建比分矩阵，比较大小球、让球 EV 是否收敛。

### 阶段 4：分市场验收

- 胜平负：继续使用 Brier / Log Loss / 校准误差。
- 大小球：按盘口线统计 ROI、EV 复算误差、总进球偏差。
- 让球：按盘口线和让受方向统计 ROI、胜盘概率偏差。

## 当前执行建议

当前不要调整下注策略，不要放开模拟舱买入。下一步应先执行：

1. 修复赛事名称国家污染。
2. 新增 `pbase_total_shrink` 候选比分矩阵，只用于审计对比。
3. 对比原始 pbase 与收缩后矩阵在已结算样本上的大小球/让球表现。
