## 1. 项目身份

- 项目名：世界杯预测。
- 项目目标：输入两支球队，自动抓取数据并预测 90 分钟赛果。
- 使用方式：只在甲方电脑本地运行。
- 主要界面：本地 Web 页面，地址默认为 `http://127.0.0.1:8765`。
- 命令行保留为调试和批处理入口。

## 5. 当前主要问题

### P0：API 模式依赖未来直接交锋或 fixture_id

当前已新增缓解方案：

- 后端新增 `/api/search-fixtures`，用于按球队搜索未来赛程。
- 后端新增 `/api/random-today-predict`，用于从当天 API-Football 赛程随机抽一场并预测。
- 后端新增 `/api/today-fixtures?scope=first_division`，用于按北京时间抓取赛前甲级联赛候选。
- 前端新增“搜索比赛”“今日甲级联赛”和“今日随机比赛”按钮。
- 若没有未来直接交锋，会给中文提示。

仍需实机验证：

- 用户本机服务重启后，在页面点击按钮验证 API 返回结果。
- 今日随机比赛依赖当天 API-Football 是否存在 fixtures。

原始问题逻辑是：

```text
输入两队
  ↓
搜索球队 ID
  ↓
查未来 head-to-head fixture
```

如果两队没有已排定的未来交锋，API 会返回空。此时程序必须让用户填写 `fixture_id`，或者提供比赛搜索/选择功能。

影响：

- 普通用户不知道 fixture_id。
- 页面看起来像“预测失败”，实际是“没有可预测的已排定比赛”。

### P0：API 报错时右侧仍保留上一次结果

已改进：

- API 预测失败时，前端会把右侧改成“预测失败”状态。
- 导出按钮会禁用。
- 数据提示区会显示中文错误说明。

剩余风险：

- 需要用户重启本地服务后才能加载新后端接口。

### P1：真实赔率覆盖不稳定

API-Football 的赔率、大小球、让球覆盖取决于赛事、时间和 bookmaker。某些比赛可能没有完整市场。

已完成：

- 明确显示哪些市场可用。
- 对缺失赔率给中文解释。
- 缺失市场返回 `NO_MARKET`。
- 数据质量不足时，API 模式研究方向自动降级为观望。
- 指定庄家报价按市场键、盘口线、方向和赔率时点结构化存储。
- 已对存量原始赔率快照进行规范化回填。

后续需要：

- 积累足量赛前快照并按时点区分早盘、即时盘、临场盘。
- 增加收盘线对比。

### P1：模型仍是 MVP

当前模型适合演示和框架验证，但真实预测能力不足。尚未接入：

- World Football Elo。
- FIFA 官方排名。
- 伤停。
- 首发。
- 小组积分和出线形势。
- 天气。
- xG。
- 球员身价和阵容深度。

最新研究结论：

- 下一阶段不应先堆复杂模型，而应先补数据质量评分和市场完整性展示。
- 研究方向判断已包含 `data_quality_score` 门槛；正式信号仍须等待 `pfinal` 验证。
- 模型 V2 建议采用 `Rating + Goals + Market Calibration` 架构。
- 世界杯小组赛需要从“必须赢”字段升级为结果收益函数和出线压力模型。
- 当前展示融合概率不是正式 `pfinal`；在校准回测完成前，API 模式不启用正式模拟信号。

### P1：历史回测仍需扩展

当前已新增赛果同步和胜平负时间切分校准审计，但还不是完成验收的历史回测系统。现在展示的研究试算 EV 仍是模型估算差异，不是已经用大量历史比赛验证的长期优势；API 模式不将其作为正式信号。

需要：

- 继续积累或批量导入合格赛前预测和赛果，至少达到 100 场独立比赛。
- 为大小球和让球建立比分分布层校准验收。
- 输出长期资金曲线和分市场表现。
- 做 CLV、Brier Score、Log Loss、ROI 的阶段性报告。

## 6. 产品边界

- 不连接真实投注平台。
- 不保证收益。
- 模拟舱仅用于纸上研究和回测。
- 国家关系、商业收益等情景变量只低权重处理。
- 正式输出坚持概率表达，不使用“稳赢”等绝对措辞。
