## 一、执行摘要

当前项目已经具备本地可运行 Web 产品雏形：用户可以输入两支球队或选择示例比赛，程序会输出胜平负概率、预期进球、高概率比分、市场概率、模型概率、模拟舱方向，并支持中文 Excel / PDF 报告导出。

但是，从“演示型 MVP”走向“可对外宣称有正期望能力的预测系统”，还存在明显距离。核心问题不是界面，也不是程序能不能跑，而是：

- 数据强度不足时，系统仍可能输出看似可信的 BUY。
- 赔率使用的是平均赔率，不是可成交赔率。
- 正期望没有经过历史回测验证。
- API 原始数据没有快照留档，无法复盘预测当时的数据环境。
- 世界杯专项变量尚未真正接入，包括小组积分、出线形势、伤停、首发、天气、Elo/FIFA 底座。

因此，当前版本适合定义为：

```text
本地可运行原型 + 自动取数演示 + 中文报告交付样机 + 正期望模拟舱框架
```

当前版本不适合定义为：

```text
已回测验证的真实正期望投注模型
```

### 最新整改进展

- 今日随机比赛已改为只抽取赛前状态且开赛时间晚于当前北京时间的比赛。
- 缺少可靠球队强度数据时，模拟舱 BUY 会强制降级为 WATCH，并在数据质量中降低评分。
- API 模式已保存 fixture、odds、team statistics、h2h 原始快照，并与预测记录关联。
- 新增最小回测模块，可录入 90 分钟比分并结算胜平负、大小球、让球模拟舱结果。
- 页面和中文报告已将模拟舱概率口径标为“独立模型概率”，避免与最终融合概率混淆。

## 五、核心算法模型

### 1. 球队基础预期进球

当前模型以双方攻防评分为基础：

```text
lambda_H_base = base_goals * safe_ratio(attack_H, defense_A)
lambda_A_base = base_goals * safe_ratio(attack_A, defense_H)
```

其中：

```text
base_goals = 1.28
safe_ratio(x, y) = clamp(x, 0.2, 2.5) / clamp(y, 0.2, 2.5)
```

### 2. 特征优势项

模型将多种因素转成主队相对客队的优势值：

```text
E_total =
  E_elo
  + E_rank
  + E_host
  + E_rest
  + E_travel
  + E_group
  + E_rotation
  + E_h2h
  + E_country
  + E_commercial
```

主要公式：

```text
E_elo = clamp((elo_H - elo_A) / 400, -1.5, 1.5) * strength_weight

E_rank = clamp((rank_A - rank_H) / 100, -1.0, 1.0) * rank_weight

E_rest = clamp((rest_days_H - rest_days_A) / 5, -1.0, 1.0) * rest_weight

E_travel = clamp((travel_km_A - travel_km_H) / 6000, -1.0, 1.0) * travel_weight

E_rotation = clamp(rotation_risk_A - rotation_risk_H, -1.0, 1.0) * rotation_weight

E_h2h = h2h_edge_H * h2h_weight
```

### 3. 预期进球修正

优势项用指数方式作用于双方预期进球：

```text
lambda_H = lambda_H_base * exp(E_total)
lambda_A = lambda_A_base * exp(-E_total)
```

再限制极端值：

```text
lambda_H = clamp(lambda_H, 0.15, 4.5)
lambda_A = clamp(lambda_A, 0.15, 4.5)
```

### 4. 泊松比分矩阵

单队进球服从泊松分布：

```text
P_H(i) = exp(-lambda_H) * lambda_H^i / i!
P_A(j) = exp(-lambda_A) * lambda_A^j / j!
```

比分概率：

```text
P_score(i, j) = P_H(i) * P_A(j)
```

当前枚举 0 到 8 球，并归一化。

### 5. 胜平负模型概率

```text
P_model(H_win) = sum P_score(i, j), i > j
P_model(draw) = sum P_score(i, j) * (1 + draw_boost), i = j
P_model(A_win) = sum P_score(i, j), i < j
```

其中：

```text
draw_boost = rivalry_intensity * draw_rivalry_weight
```

### 6. 市场概率去水

三项欧赔：

```text
raw_H = 1 / odds_H
raw_D = 1 / odds_D
raw_A = 1 / odds_A

P_market(k) = raw_k / (raw_H + raw_D + raw_A)
```

两项市场：

```text
P_market(side) = (1 / odds_side) / sum(1 / odds_all_sides)
```

### 7. 最终概率融合

```text
P_final(k) =
  (1 - market_weight) * P_model(k)
  + market_weight * P_market(k)
```

当前默认：

```text
market_weight = 0.45
```

注意：当前模拟舱 EV 使用的是模型概率，不是最终融合概率。

### 8. 模拟舱 EV

胜平负：

```text
EV = P_model(selection) * odds - 1
edge = P_model(selection) - P_market(selection)
```

BUY 条件：

```text
EV > 0
edge >= min_edge
data_quality_score >= min_quality
```

大小球和让球会根据亚洲盘结算规则计算每个比分下的收益，再对比分矩阵求期望。

### 9. 数据质量评分

当前数据质量：

```text
data_quality_score =
  0.20 * fixture_certainty
  + 0.25 * odds_completeness
  + 0.15 * bookmaker_quality
  + 0.20 * team_rating_availability
  + 0.10 * context_availability
  + 0.10 * lineup_availability
```

等级：

```text
>= 0.80  高
>= 0.60  中
>= 0.40  低
< 0.40   很低
```

## 八、深度判断：项目真正的难点

这个项目表面是“输入两队给预测”，本质上是三类系统的叠加：

```text
数据系统 + 概率模型系统 + 风控/回测系统
```

当前项目已经把产品外壳和第一版概率链路做出来了，但真正决定价值的是后两步：

1. 数据是否足够强。
2. 概率是否经过校准。
3. EV 是否用真实可成交赔率计算。
4. 预测是否能赛后结算。
5. 回测是否证明长期不亏或有优势。

如果这些没有完成，程序仍然可以作为研究工具，但不应被包装成“能下注赚钱”的模型。
