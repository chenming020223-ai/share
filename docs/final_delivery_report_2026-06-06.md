## 一、交付结论

本项目当前可以作为“可长期使用的本地研究模拟系统”交付。

可交付内容包括：

- 本地网页工作台；
- API-Football 自动取数；
- 单场赛前分析；
- 多场批量分析；
- 数据处理复核；
- 模型审计；
- 赛后复盘；
- 中文 Excel / PDF 报告导出；
- 本地 SQLite 长期留档；
- 双击启动和停止；
- 每日批量建库入口；
- 模拟舱研究账本和赛果同步。

但当前不能作为“已经验证可盈利的正式下注模型”交付。

原因是：

- `pfinal` 尚未批准；
- 正式 EV 仍关闭；
- 合格赛前已结算样本只有 `15 / 100`；
- 大小球和让球的比分矩阵层仍需单独校准；
- 当前 EV 候选复盘仍出现高 EV 亏损，说明不能放开正式资金占用。
- 基础胜率已加入球队强度先验和对手强度校正，但仍需要后续时间切分样本验证。

因此，本版本的正确定位是：

```text
可以落地使用，用于赛前研究、数据积累、模型审计、复盘和模拟舱观察；
不能宣传为稳定盈利模型，不能自动产生正式下注指令。
```

## 三、已落地功能

### 3.1 赛前预测工作台

- 输入球队或比赛 ID；
- 搜索 API-Football 赛前比赛；
- 今日甲级联赛赛程选择；
- 随机抽取今日赛前比赛；
- 指定 fixture_id 批量分析；
- 只允许尚未开赛比赛形成预测快照。

### 3.2 数据抓取

- 默认深度模式；
- 获取 fixture、league、team、venue；
- 获取双方近 10 场比赛；
- 对近期比赛做时间衰减和对手强度校正；
- 国家队、U21 和友谊赛融合内部球队强度先验，降低短期样本误导；
- 双方各至少 5 场有效 90 分钟比赛才具备研究准入；
- 深度模式补充历史比赛技术统计和事件；
- 获取优先级庄家全场盘口；
- 保存原始 API 快照。

### 3.3 盘口规则

默认庄家优先级：

```text
Pinnacle > Bet365 > Betfair > SBO > 10Bet > 1xBet
```

当前规则：

- 胜平负必须同一家完整全场 1X2；
- 大小球必须同一家、同盘口线、成对全场赔率；
- 让球必须同一家、同盘口线、成对全场赔率；
- 不跨庄家拼盘口；
- 不使用半场、角球、卡牌等非全场市场；
- 水位异常会被排除；
- 每个市场记录实际使用庄家。

### 3.4 概率与 EV 口径

胜平负：

```text
EV = pbase(selection) * odds - 1
```

大小球 / 让球研究审计：

```text
research_EV = 盈利注权重 * (odds - 1) - 亏损注权重
```

大小球和让球当前只展示逐比分结算的研究审计值。比分分布层未完成专项校准前，`paper_EV`、`p_adj`、`shrink_k` 与正式 EV 均不开放。

页面和报告已经区分：

- 胜平负：模型胜率；
- 大小球 / 让球：正收益概率；
- 正式 EV：当前未开放。

### 3.5 模拟舱

资金规则已落地：

- 启动资金默认 1000；
- 动态均注按资金五等分；
- 盈利只按 50% 纳入后续注额；
- 单场多市场有占用上限；
- 未通过正式 EV 时资金占用为 0。

当前模拟舱状态：

```text
研究锁定；不产生正式 PAPER_BUY；不占用资金。
```

## 七、正式上线还差什么

### 阶段 A：数据样本

目标：

- 合格赛前已结算样本达到 100 场以上；
- 每场都有明确赔率时点；
- 每场都有 90 分钟赛果；
- 每场都有 `pbase` 和 `qmkt`。

### 阶段 B：胜平负校准

目标：

- 按时间切分开发、校准、验证；
- 验证 `pshr` 是否优于 `pbase`；
- 验证 `pshr` 是否接近或优于市场基准；
- 通过后再定义 `pfinal`。

### 阶段 C：大小球 / 让球校准

目标：

- 单独验收比分矩阵；
- 按盘口线统计 EV 与真实盈亏；
- 校准总进球层；
- 校准让球线方向；
- 通过后才允许进入模拟舱资金规则。

### 阶段 D：交付公网下载版

目标：

- 使用新的 API-Football 密钥；
- 设置访问口令；
- 配置持久磁盘；
- 打包用户使用说明；
- 提供可分享访问地址或可下载安装包。
