# 网页界面使用说明

日期：2026-05-27

## 启动网页

首次使用先双击项目根目录的 `setup_worldcup_predictor.command`。

日常使用双击 `start_worldcup_predictor.command`。它会在后台启动本地网页服务，并自动打开：

```text
http://127.0.0.1:8765
```

启动成功后可以关闭弹出的终端窗口，网页服务会继续运行。需要停止时双击：

```text
stop_worldcup_predictor.command
```

只有排查问题或开发调试时，才建议在项目目录手动运行：

```bash
python3 -m worldcup_predictor.web_server
```

健康检查：

```text
http://127.0.0.1:8765/healthz
```

如果端口被占用：

```bash
python3 -m worldcup_predictor.web_server --port 8788
```

也可以通过 `.env` 配置：

```text
WORLDCUP_WEB_HOST=127.0.0.1
WORLDCUP_WEB_PORT=8765
WORLDCUP_DB_PATH=storage/worldcup_predictor.sqlite3
API_FOOTBALL_KEY=你的_api_key
API_FOOTBALL_RETRIES=3
```

## 数据入口

- 页面仅提供真实赛事分析入口，使用 API-Football 自动抓取比赛和赔率。
- 页面初次打开时不自动生成预测；只有选择真实赛程或输入比赛 ID 后提交，才保存新的预测运行。
- 页面默认使用“深度模式”：会在基础比赛、赔率、近期赛果之外，补抓双方近 10 场技术统计和事件数据。
- 如遇 API 额度紧张，可在“抓取深度”中临时切换为快速模式。

## 页面输出

- 页面以“预测工作台 / 批量赛事池 / 数据处理 / 模型审计 / 历史报告”五个区域组织信息。
- 展示融合概率（非 `pfinal`）。
- `pbase` 基础概率。
- `qmkt` 市场去水概率。
- 预期进球。
- 最可能比分。
- 胜平负、大小球、让球模拟舱方向。
- 启动资金、均注金额、占用资金、期望资金。
- 本地保存的预测 Run ID。
- 中文 Excel / PDF 报告导出按钮。

## 数据处理复核

- “数据处理”展示本场抓取流程、双方最近 10 场中可纳入的有效 90 分钟比赛，以及场均积分、场均进失球、攻防评分等特征。
- 深度模式会额外展示技术统计覆盖、场均 xG、射门、射正、控球、红牌和点球。
- 近期比赛以时间顺序绘制双方进失球线图，便于检查输入样本是否与模型输出方向一致。
- 模拟舱图表改为资金与期望资金曲线，优先用于观察长期纸上资金表现，而不是单场积分走势。
- API 原始队名仅在这一审计区域随中文名附带显示，供翻译核定和问题追踪。
- 若本场只保存了一个赛前赔率快照，页面会明确说明不能据此绘制真实赔率走势；赔率时间序列应在持续采样后开放。

## 中文展示标准

- 比赛标题显示中文球队名，例如“墨西哥 vs 美国”。
- 比赛信息显示具体联赛名称，例如“国际友谊赛”或“国际足联世界杯”。
- API 时间统一转换为北京时间，例如“2026-06-12 08:00 北京时间”。
- 赛程搜索结果、概率表、市场表和中文报告都优先使用项目受控中文名录；未核定的名称显示 API 英文原名，避免无法识别，ID 继续保留供核验。

## API 辅助功能

页面提供三个赛事入口按钮：

- 搜索比赛：按球队搜索 API-Football 已排定赛程，选择后自动填入比赛 ID。
- 今日甲级联赛：抓取北京时间今天尚未开赛、被项目中文名录识别为足球甲级联赛的比赛列表；选择一场后点击“生成赛前分析”。
- 今日随机比赛：从当天 API-Football fixtures 中随机抽一场并直接预测。
- 批量分析 10 场：默认分析今日甲级联赛前 10 场；也可以在“批量比赛 ID”中输入多个 fixture_id，用逗号、空格或换行分隔，系统会按指定 ID 批量分析。

搜索结果中的每一场比赛都提供“单场分析”和“加入批量”两个动作：前者填入单场比赛 ID，后者加入批量 ID 输入框。

批量分析完成后，页面会自动切换到“批量赛事池”主页面，显示批量总览，包括成功场次、信号数、观望数、失败数、预计资金；每场比赛会直接展示预测方向、模拟舱方向、数据质量、盘口覆盖、EV 状态、庄家和资金结果。点击“查看单场”可回到该场完整分析页面。

每次批量分析都会生成本地批次编号并保存到 SQLite。刷新网页后，打开“批量赛事池”可以看到最近批次列表，点击“恢复批次”即可重新查看该批次的筛选、排序、组合预案和单场入口；恢复批次只读取本地数据库，不重新消耗 API-Football 请求。

批次历史支持按名称、备注、日期和 fixture_id 搜索，并按每页 5 个批次分页展示。每个批次可以直接填写“批次名称”和“备注”，用于记录当时的筛选条件、异常盘口、复盘结论或甲方确认信息。名称和备注会写入本地 SQLite，并同步写回该批次完整快照。

批量结果可按范围、质量和排序方式筛选：

- 范围：全部、只看信号、组合预案、只看观望、市场缺失、失败记录。
- 质量：全部质量、高质量、可复核。
- 排序：推荐优先、质量优先、EV 优先、时间优先。

批量赛事池还支持进一步细筛：

- 关键词：可按球队、比赛 ID、联赛、推荐方向、推荐原因搜索；常用缩写如英超、意甲、法乙也可匹配官方中文联赛名。
- 联赛：只看某一个联赛。
- 庄家：按实际使用庄家筛选；若胜平负、大小球、让球来自不同庄家，页面会分别显示。
- 盘口覆盖：全部盘口、三项完整、部分可用、盘口缺失。
- 开赛时段：18:00 前、18:00-23:00、23:00 后/凌晨。

当前批量资金仍按“逐场独立评估”展示，尚未按同一批次顺序扣减资金；页面提供“研究组合预案”，按推荐优先、质量和保守 EV 排序，单批建议占用不超过启动资金 50%，并提示同联赛或同市场集中风险。正式资金组合策略需要在批量赛事池稳定后再开启。

如果两队没有未来直接交锋，页面会提示选择赛程或填写比赛 ID。
球队输入框可输入已映射的中文球队名；程序会在调用 API-Football 前反查为 API 可识别名称。
API-Football 不直接提供官方中文名字段；当前页面对已收录名称优先中文展示，未收录名称保留 API 英文原名，新增赛事的最终中文译名可由甲方验收后补充确认。

## 本地留档

每次网页预测都会保存到 SQLite：

```text
storage/worldcup_predictor.sqlite3
```

健康检查接口：

```text
http://127.0.0.1:8765/api/health
```

API 连接检查：

```text
http://127.0.0.1:8765/api/api-status
```

最近预测记录：

```text
http://127.0.0.1:8765/api/recent-predictions
```

“历史报告”页面会直接展示每条记录的最高概率预测、模拟舱推荐/观望摘要、资金变化、数据质量和实际使用庄家；Excel/PDF 是进一步复核入口，不再是查看结论的唯一方式。

## 每日批量建库

项目已加入后台批量建库入口，用于每天自动积累赛前快照和后续回测样本。

手动执行：

```bash
python3 -m worldcup_predictor.batch_collect --scope first_division --mode batch
```

也可以双击：

```text
daily_batch_collect.command
```

如需安装 macOS 每天 08:30 自动运行，双击：

```text
install_daily_batch.command
```

批量建库默认范围为北京时间今日甲级联赛；可用 `.env` 调整：

```text
WORLDCUP_BATCH_SCOPE=first_division
WORLDCUP_BATCH_MAX_FIXTURES=30
WORLDCUP_DEEP_MATCH_LIMIT=10
```

## 重要说明

模拟舱只用于纸上回测和研究，不连接真实投注平台，也不保证收益。

## API 证书问题

如果页面提示 `CERTIFICATE_VERIFY_FAILED`，先双击 `stop_worldcup_predictor.command` 停止本地服务，再双击 `start_worldcup_predictor.command` 重启。

开发调试时也可以在终端中停止前台服务再启动：

```bash
Control + C
python3 -m worldcup_predictor.web_server
```

项目会优先使用 `certifi` 证书包。如果仍失败，请参考 `docs/troubleshooting.md`。

## 中文报告导出

每次预测完成后，可点击“导出 Excel”或“导出 PDF”。报告包含：

- 90 分钟胜平负概率。
- 具体联赛名称和北京时间。
- 模型概率和市场概率。
- 预期进球和可能比分。
- 正期望模拟舱方向。
- 资金占用和期望资金。
- 数据处理特征摘要；Excel 额外列出近期比赛明细。
- 数据提示和风险边界。
