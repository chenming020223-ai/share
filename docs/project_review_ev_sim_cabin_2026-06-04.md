# 世界杯预测项目审查与模型改进分析报告

日期：2026-06-04  
对象：世界杯预测本地 Web 系统、EV 计算链路、模拟舱、数据与模型治理  
口径：仅评估 90 分钟赛果；不连接真实投注平台；不承诺收益

## 1. 总体结论

当前项目已经可以交付为“足球比赛预测研究与纸上复盘系统”，但不能交付为“已验证正期望投注系统”。

核心原因不是页面功能不足，而是概率模型尚未完成正式校准。当前 EV 主要使用 `pbase` 基础概率进行研究试算，而 `pbase` 尚未证明优于市场基准，也不是正式可执行概率 `pfinal`。因此，系统现在最正确的策略是继续保持正式 EV 关闭，只把 EV 当作模型分歧和复盘诊断工具。

- 本地 Web 页面可运行，默认地址为 `http://127.0.0.1:8765`。

## 3. 概率模型审查

### 3.1 当前概率对象

| 概率 | 含义 | 当前用途 | 是否能进正式 EV |
|---|---|---|---:|
| `pbase` | 基础进球/比分模型概率 | 研究展示、校准输入 | 否 |
| `qmkt` | 优先庄家去水概率 | 市场基准 | 否 |
| `pshr` | 时间切分收缩候选概率 | 审计候选 | 否 |
| `pfinal` | 通过校准和回测审批的执行概率 | 未来正式信号 | 是 |

当前页面展示的融合概率不是 `pfinal`，不能用于正式下注。

### 3.2 当前验收结论

模型验收接口显示：

- 状态：`INSUFFICIENT_DATA`，校准样本不足。
- `pfinalStatus = not_approved`。
- 验证区间中，`pshr` 未显著优于市场基准，甚至落后市场基准：
  - Log Loss 差 `+0.1152`
  - Brier 差 `+0.0732`

这说明当前基础模型和候选收缩模型还没有足够证据证明可以战胜市场。

### 3.3 明显问题

1. `pbase` 对冷门概率偏乐观。
   复盘中多个超高 EV 来自冷门方向，例如高赔率客胜。这类信号大多不是市场错误，而是模型高估冷门概率。

2. 模型分歧组表现很差。
   2026-06-03 复盘中，“模型分歧暂停”组 6 场仅命中 1 场，平均 LogLoss 约 `1.191`，明显劣于研究观察组。

3. 市场信息没有被充分用作校准约束。
   当前已做去水概率 `qmkt` 和模型分歧闸门，但尚未形成通过时间切分验证的 `pfinal`。

4. 小样本下不应解读 ROI。
   15 场合格样本、3 场验证区间样本都太少，不足以判断长期优势。

## 4. EV 计算审查

### 4.1 胜平负 EV

当前研究 EV：

```text
research_EV = pbase_i * odds_i - 1
```

展开为：

```text
research_EV = pbase_i * (odds_i - 1) - (1 - pbase_i)
```

数学路径正确，但概率输入不具备正式资格。主要风险是 `pbase` 偏差会被赔率放大，特别是高赔率冷门。

### 4.2 大小球和让球 EV

当前大小球和让球使用比分矩阵结算期望：

```text
research_EV = sum(P_score(s) * net_return(s, line, odds))
```

已能处理：

- 全赢
- 半赢
- 走水
- 半输
- 全输

这是比早期版本明显更正确的方向。

但仍有问题：

1. `model_probability` 使用“正收益概率 positive”，不完全等价于亚洲盘真实胜率。
   对 2.25、2.75、-0.25、-0.75 等盘口，半赢半输和走水会影响真实价值，仅用正收益概率与市场概率比较会失真。

2. 大小球高 EV 亏损明显。
   2026-06-03 复盘中，大小球已结算净亏约 `-2.36`，说明进球分布层需要专项校准。

3. 没有分盘口线回测。
   2.25、2.5、2.75、3.25 的表现可能完全不同，不能混在一起看。

### 4.3 保守 EV

当前保守 EV 近似为：

```text
conservative_EV = EV - probability_discount * odds
```

它能压制高赔率，但比较粗糙。更合理的方式应该按以下维度动态折扣：

- 联赛级别
- 数据质量
- 市场类型
- 盘口线
- 赔率区间
- 模型与市场分歧程度
- 近期样本数量

## 5. 模拟舱审查

### 5.1 当前正确点

- 正式 EV 未开放时，资金占用为 0。
- 不会把研究候选伪装成真实 BUY。
- 已具备动态五等分资金策略雏形。
- 已有单场多市场总暴露上限。
- 已支持账本字段和赛后结算字段。

### 5.2 当前不足

1. 批量分析还不是组合级资金系统。
   当前更多是逐场评估，还缺少按开赛时间排序的资金扣减、结算、再分配。

2. 缺少每日总风险限制。
   未来开放正式模拟时，需要每日最大占用、最大场次、最大亏损暂停线。

3. 缺少相关性控制。
   同联赛、同时间段、同类型市场的信号可能高度相关，不能按独立事件处理。

4. 高赔率冷门没有独立限额。
   当前高赔率冷门虽然多数被闸门拦住，但未来若开放正式 EV，应设置更低注额或单独审批。

5. 模拟舱状态语义需要进一步清晰。
   建议分为：
   - `MODEL_CANDIDATE`：模型候选，仅说明模型看到差异。
   - `RESEARCH_WATCH`：研究观察，不占用资金。
   - `PAPER_BUY`：正式纸上买入，占用模拟资金。
   - `NO_MARKET`：盘口缺失。
   - `SUSPENDED`：分歧或质量异常暂停。

## 7. 下一阶段执行路线

### 阶段 A：数据优先

目标：把可校准样本从 15 场提升到至少 100 场。

任务：

1. 每天固定时间跑批量建库。
2. 只纳入赛前快照、全场盘口、有效赔率时点、90 分钟赛果。
3. 按庄家、联赛、盘口类型、盘口线保存分层字段。
4. 增加收盘赔率字段，为 CLV 做准备。
5. 对未取得盘口的比赛只保留模型观察，不进入 EV 样本。

### 阶段 B：概率校准

目标：形成候选 `pfinal`。

建议公式：

```text
pfinal_raw = w_market * qmkt + w_model * pbase + w_context * pcontext
```

其中：

```text
w_market + w_model + w_context = 1
```

权重不得手工拍脑袋，应通过时间切分数据拟合，并按验证集 Brier、LogLoss、校准误差审批。

### 阶段 C：市场专项回测

胜平负、大小球、让球必须分开验收：

- 胜平负：三分类概率校准。
- 大小球：总进球分布校准，按盘口线分层。
- 让球：净胜球分布校准，按让球线分层。

### 阶段 D：模拟舱组合化

建立真实纸上资金曲线：

```text
初始资金 = 1000
基础单注 = 当前可用资金 / 5
盈利再投入 = 利润 * 50%
单场最大暴露 <= 当前资金 * 40%
单日最大暴露 <= 当前资金 * 60%
单市场最大暴露 <= 当前资金 * 25%
连续亏损 3 笔降档
回撤 >= 20% 暂停新信号
```
