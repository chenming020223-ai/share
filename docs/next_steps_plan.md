# 后续计划

日期：2026-05-22  
目标：把当前可演示 MVP 推进到可稳定使用的本地交付版。

## 总体策略

先解决“能不能顺畅使用”，再提升“预测是否更准”。

当前优先级：

1. API 使用体验。
2. 数据完整性提示。
3. 中文展示标准。
4. 报告和交付体验完善。
5. 数据快照和回测基础。
6. 模型增强。

## 里程碑 1：API 输入体验修复

目标：用户不需要自己查 fixture_id，也不会被英文错误困住。

当前状态：已完成第一版代码实现，API 自动重试和中文错误分类已补齐，待用户本机重启服务后实机验证。

### 任务

- 新增 fixture 搜索接口：
  - 输入球队 A、球队 B。
  - 返回未来相关比赛列表。
  - 支持按一队搜索近期/未来赛程。
  - 支持用户从页面点选比赛。
- 页面增加“搜索比赛”按钮。
- 页面增加“今日随机比赛”按钮。
- 搜索结果显示：
  - fixture_id。
  - 主队、客队。
  - 北京时间开赛时间。
  - 官方中文赛事名称。
  - 状态。
- API 预测失败时：
  - 清空当前结果区，或标记为“上一次结果”。
  - 错误提示改成中文。
  - SSL EOF、证书、限流、密钥错误分类型展示。
- 若没有未来直接交锋：
  - 提示“API-Football 未找到两队已排定比赛，请从比赛列表选择或填写 fixture_id。”

### 验收

- 输入 `Mexico`、`USA` 时，即使没有未来交锋，也能得到清楚中文提示。
- 填入有效 fixture_id 时能正常预测。
- API 失败不会让旧结果误导用户。
- 点击“今日随机比赛”后，如果当天有 API-Football fixtures，可以随机抽一场并输出预测。
- 网页和报告里的球队、联赛、开赛时间统一使用中文展示字段。

## 里程碑 2：市场数据完整性

目标：让用户知道当前能分析哪些市场，哪些市场缺数据。

当前状态：第一版已完成。

### 任务

- 增加市场状态卡片：
  - 胜平负：可用/缺失。
  - 大小球：可用/缺失。
  - 让球：可用/缺失。
- 对每个市场显示：
  - 使用的盘口线。
  - 赔率来源 bookmaker 数量。
  - 是否参与模拟舱。
- 缺少赔率时：
  - 不生成 `BUY`。
  - 报告中写明缺失原因。
- 数据质量不足时：
  - API 真实预测将 `BUY` 降级为 `WATCH`。
  - 中文报告和网页展示质量分。

### 验收

- 没有大小球赔率时，页面显示“大小球市场缺失”，而不是空白或误判。
- 中文报告能同步写出市场缺失说明。
- 单元测试覆盖市场完整、缺失、不完整和质量门槛降级。

## 里程碑 4：模型增强第一期

目标：让球队强度不只依赖 API-Football 的赛季统计。

理论研究：`docs/model_optimization_research.md`

模型 V2 方向：`Rating + Goals + Market Calibration`。

### 任务

- 先新增数据质量评分：
  - fixture 确定性。
  - 赔率完整性。
  - bookmaker 数量。
  - 球队评分可用性。
  - 世界杯情景数据可用性。
- 将模拟舱 BUY 规则升级为：
  - `EV > 0`。
  - `model_probability - market_probability >= min_edge`。
  - `data_quality_score >= min_quality`。
- 接入或导入 World Football Elo。
- 接入或导入 FIFA 官方排名。
- 建立球队名称映射表。
- 模型中加入：
  - Elo 差。
  - FIFA 排名差。
  - 近期状态。
- API 无法提供统计时，用 Elo/FIFA 兜底。

### 验收

- 数据不足时，系统能明确降级为 WATCH 或低置信观望。
- 国家队比赛即使没有完整赛季统计，也能输出更合理的强度基础。
- 报告解释中显示 Elo/FIFA 对预测方向的影响。
