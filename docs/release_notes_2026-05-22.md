# 世界杯预测交付更新记录

日期：2026-05-22

## 2026-05-26 模型治理更新

- 新增 `docs/model_adoption_spec_v1.md`，冻结 `pbase -> qmkt -> pshr -> pfinal -> 正式 EV` 的准入边界。
- 页面与报告将当前融合输出明确标记为“展示融合概率（非 pfinal）”，基础模型与市场去水概率分别标记为 `pbase` 与 `qmkt`。
- 新增正式 EV 治理闸门：在 `pshr/pfinal` 尚未通过时间切分校准与回测验证前，API 模式保留候选 EV 供复核，但所有候选信号降级为观望，模拟资金占用为零。
- 新增结构化 `market_quotes` 表，存储 Pinnacle 全场盘口的市场键、方向、赔率、去水概率、赔率时点与模型版本；现有数据库已回填 `236` 条报价。
- 新增胜平负时间切分校准审计 `calibration.py`，按开发/校准/验证区间比较 `pbase`、`qmkt` 与候选 `pshr`，不会自动开启 `pfinal`。
- 页面新增“校准与回测准入”面板和“同步赛果”按钮；已同步 `2` 场合格赛前预测的 90 分钟结果。
- 手工填写已开赛比赛 ID 时不再允许生成预测快照；既有赛后记录会被标记并隔离于校准样本之外。
- 新增“今日甲级联赛”赛前列表接口与页面按钮，按北京时间筛选赛前比赛，并补充哈萨克斯坦、亚美尼亚、加蓬当日候选的中文显示名录。
- 报告和 API payload 区分“指定庄家 Pinnacle”与“已取得盘口庄家”；没有赔率时不再给出易误读的取得庄家信息。
- 非执行动作的明细金额归零：`NO_MARKET` 或治理闸门降级后的 `WATCH` 不再携带可被误读为下注的金额。
- 页面真实验证运行 `56`：哈萨克斯坦足球甲级联赛“阿斯塔纳二队 vs 汗腾格里”，2026-05-26 22:00 北京时间；未取得 Pinnacle 盘口，正确输出市场缺失和占用 `0` 元。

## 2026-05-24 模型风控更新

- 胜平负、大小球、让球的模拟舱输出从“买入”降温为“模拟信号”，强调仅用于纸上研究。
- 默认优势门槛从 3% 提高到 8%，并新增基础 EV 5% 门槛。
- 新增保守 EV 闸门：模型概率默认扣减 5 个百分点后，EV 仍需大于 0。
- 大小球和让球只接受同一个 bookmaker、同一个盘口线的成对赔率。
- 胜平负只接受同一个 bookmaker 提供完整主胜/平局/客胜三项赔率后再参与聚合。
- 两项盘口新增水位异常校验：隐含概率和必须在 98% 到 125% 之间，否则直接排除。
- 针对“大 2.5 赔率 4.85、EV 200%+”这类异常盘口，程序会判为市场无效，不再生成模拟信号。

## 本次更新重点

- API-Football 请求增加自动重试，默认重试 3 次。
- SSL EOF、证书错误、密钥错误、限流错误、接口错误改为中文分类提示。
- 新增 API 连接检查接口：`/api/api-status`。
- “搜索比赛”改为后台请求体传递 API Key，避免密钥出现在地址栏。
- 新增双击启动脚本：`start_worldcup_predictor.command`。
- 新增首次依赖安装脚本：`setup_worldcup_predictor.command`。
- 新增甲方交付说明：`交付说明.md`。
- 保留中文 Excel / PDF 报告导出。
- 今日随机比赛只抽取赛前状态且开赛时间晚于当前北京时间的比赛。
- API 模式新增原始数据快照保存，预测记录关联快照编号。
- 缺少可靠球队强度数据时，模拟舱 BUY 强制降级为 WATCH。
- 页面和报告将模拟舱概率口径标为“独立模型概率”。
- 新增最小回测模块，支持录入 90 分钟比分后结算胜平负、大小球、让球。
