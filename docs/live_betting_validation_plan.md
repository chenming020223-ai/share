# 实盘下注系统验证与上线计划

日期：2026-06-06  
状态：实盘验证基线  
结论：当前项目不能被声明为“已验证可实盘下注系统”，但已经升级为可持续验证的本地研究与纸上验证系统。

## 1. 交付定位

当前系统允许交付为：

- 本地赛前数据抓取与预测分析系统；
- 研究 EV 与模型分歧审计系统；
- 批量建库、赛果同步、赛后复盘和中文报告系统；
- 实盘准入判定系统。

当前系统不得交付为：

- 已验证盈利的真实下注系统；
- 可直接用真实资金自动跟单的系统；
- 已批准 `pfinal` 的正式 EV 执行模型。

## 2. 实盘准入硬门槛

真实资金只能在所有门槛通过后进入人工审批，不允许自动放开。

| 闸门 | 最低门槛 | 当前原则 |
|---|---:|---|
| 合格赛前已结算样本 | 100 场 | 必须预测和赔率均早于开赛，且有 90 分钟赛果 |
| 独立比赛数量 | 100 场 | 同一 fixture 只取一条合格记录 |
| 校准区间样本 | 20 场 | 按开赛时间切分，不随机打散 |
| 验证区间样本 | 20 场 | 只用于时间外评估 |
| pfinal 状态 | approved | 未批准前正式 EV 一律为空 |
| 大小球 / 让球专项校准 | 必须通过 | 不可借用胜平负校准结论 |
| 纸上下注已结算 | 50 笔 | 必须来自正式纸上账本，不使用主观回填 |
| 纸上 ROI | ≥ 2% | 同时观察赔率区间和市场分组 |
| 最大回撤 | ≤ 20% | 超过即暂停新信号 |

## 3. 概率与 EV 身份

当前系统保留四层概率身份：

```text
pbase  = 基础模型概率，只用于研究
qmkt   = 优先庄家去水市场概率，只作为市场基准
pshr   = 时间切分校准候选，样本不足时不可执行
pfinal = 通过验收和人工审批后的执行概率
```

正式 EV 只能使用：

```text
formal_EV = expected_net_return(pfinal, settlement_object, executable_odds)
```

当前页面中的研究 EV 使用 `pbase` 或比分矩阵，只能用于发现模型与盘口分歧，不能驱动真实资金。

## 4. 为什么现在还不能实盘

当前主要缺口：

1. 合格赛前已结算样本不足，时间切分校准没有达到审批规模。
2. `pfinal` 尚未批准，正式 EV 仍关闭。
3. 大小球和让球还没有完成独立的比分分布层校准。
4. 纸上账本没有形成足够的已结算正式纸上买入样本。
5. 当前系统尚未经过连续周期的低回撤验证。

## 5. 下一步执行路径

### 阶段 A：数据闭环

- 每天后台批量抓取赛前比赛；
- 保存 fixture、赔率、盘口线、庄家、抓取时间和模型版本；
- 每天同步完赛 90 分钟赛果；
- 排除赛后生成快照和缺少赔率时点的记录。

### 阶段 B：模型校准

- 胜平负继续推进 `pbase/qmkt/pshr` 时间切分比较；
- 大小球按盘口线建立比分矩阵结算回测；
- 让球按亚洲盘拆半注结算回测；
- 分市场、分赔率区间、分模型分歧区间输出 Brier、Log Loss、ROI 和最大回撤。

### 阶段 C：纸上验证

- 只在 `pfinal_candidate` 通过研究闸门时写入纸上账本；
- 每笔记录下注时赔率、盘口线、方向、stake、赛果和盈亏；
- 连续结算不少于 50 笔后再评估 ROI 和回撤；
- 高赔率冷门、大小球高 EV、让球深盘单独分组审计。

### 阶段 D：人工审批

- 输出完整校准报告；
- 冻结 `pshr -> pfinal` 公式；
- 明确资金管理规则、最大单日风险和止损规则；
- 审批通过后，系统仍只输出“模拟实盘信号”，不连接真实投注平台。

## 6. 当前新增实盘准入判定

系统已新增 `/api/live-readiness`：

- 返回是否允许真实资金；
- 返回所有未通过闸门；
- 返回纸上账本、样本、盘口和模型验收状态；
- 当前固定 `canUseRealMoney = false`，即使未来达到待审批条件，也必须由人工确认后再修改代码。

这条规则是交付底线：宁愿少给信号，也不能给未经验证的实盘信号。
