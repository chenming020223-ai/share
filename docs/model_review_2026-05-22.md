# 世界杯预测模型审查

日期：2026-05-22  
审查范围：模型计算链路、API 数据输入、赔率融合、模拟舱 BUY 规则、数据质量门槛。  
审查结论：当前模型可以作为本地演示和流程验证版本，但还不能按“真实正期望模型”对外交付。2026-05-22 已完成第一批 P0 安全整改：赛前过滤、球队强度缺失降级、API 快照、最小回测基础。

## 一、核心结论

当前系统的工程流程已经成型：

```text
API/示例数据
  -> 球队画像
  -> 预期进球
  -> 泊松比分矩阵
  -> 胜平负概率
  -> 市场赔率去水
  -> 模型/市场融合
  -> 模拟舱 EV 与 BUY/WATCH
```

最大问题不是程序跑不通，而是“模型置信度表达偏乐观”。尤其在 API 模式下，如果球队统计缺失，系统会回退到默认球队画像，但只要赔率市场完整，数据质量分仍可能超过 BUY 门槛。

## 二、审查发现

### P0：球队真实评分缺失时，数据质量门槛仍可能放行 BUY

位置：

- `worldcup_predictor/auto_predict.py:153-155`
- `worldcup_predictor/web_server.py:226-233`
- `worldcup_predictor/data_quality.py:68-75`

问题：

API 模式如果拿不到球队统计，`_profile_from_api` 会返回默认 `TeamProfile`。这意味着双方 Elo、FIFA 排名、进攻、防守等核心能力都接近默认值。可是数据质量评分中，球队评分缺失只被记为 `0.45`，如果赔率完整、公司数量足够，整体分数仍可能达到中等质量。

本地验证示例：

```text
fixture 确定 + 三类市场完整 + bookmaker 足够
team_rating_score = 0.45
context_score = 0.75
lineup_score = 0.0
最终数据质量分 = 76.5%
```

风险：

系统可能在“真实球队强度不足”的情况下仍给出 BUY，甲方会误以为这是强模型输出。

整改状态：

- 已完成第一版：API 模式下只要缺少可靠球队统计，模拟舱 BUY 会降级为 WATCH，数据质量分也会被压低到门槛以下。
- 下一步仍需接入 Elo/FIFA 静态底座，作为 API 统计缺失时的兜底。

### P0：今日随机比赛可能抽到已结束或进行中的比赛

位置：

- `worldcup_predictor/web_server.py:285-290`

问题：

当前“今日随机比赛”只判断 fixture 有 ID 且双方球队存在，没有过滤比赛状态。API-Football 的 date fixtures 可能包含未开始、进行中、已结束比赛。

风险：

可能对已经开赛或已经结束的比赛生成“预测”，污染模拟舱结果，也不符合赛前预测产品逻辑。

整改状态：

- 已完成第一版：只允许 `NS`、`TBD`、`PST` 等赛前状态进入随机池，并要求 kickoff 晚于当前北京时间。
- 如果今天没有赛前比赛，会返回中文提示。

### P1：模拟舱用“原始模型概率”计算 EV，但页面同时展示“最终融合概率”

位置：

- `worldcup_predictor/predictor.py:35-37`
- `worldcup_predictor/betting.py:93-111`
- `worldcup_predictor/betting.py:130-148`
- `worldcup_predictor/betting.py:169-187`

问题：

页面展示三类概率：模型、市场、最终。最终概率是模型和市场融合后的结果。但模拟舱 BUY 使用的是原始模型概率和赔率计算 EV，并不使用最终融合概率。

这在理论上可以成立，因为寻找正期望本来就要用模型对市场定价的偏差。但如果界面不解释，用户会以为 BUY 是根据“最终概率”得出的。

风险：

当市场权重较高时，最终概率和 BUY 方向可能看起来不一致。

建议整改：

- 已完成第一版：页面和报告将模拟舱概率口径标为“独立模型概率”。
- 已完成第一版：报告边界写明 EV 依据独立模型概率，不依据最终融合概率。
- 或新增参数 `ev_probability_source = model/calibrated`，默认用校准后概率。

### P1：赔率按多家公司简单平均，EV 使用的是“合成赔率”

位置：

- `worldcup_predictor/market.py:80-90`
- `worldcup_predictor/betting.py:97-99`
- `worldcup_predictor/betting.py:132-135`
- `worldcup_predictor/betting.py:171-174`

问题：

当前会把不同 bookmaker 的赔率简单平均，然后用平均赔率计算 EV。这个平均赔率不是实际可成交赔率。

风险：

模拟舱的 EV 可能高估或低估真实可下注机会。

建议整改：

- 保存 bookmaker 名称和每家公司赔率。
- 模拟舱用“可获得最佳赔率”或“指定 bookmaker 赔率”计算 EV。
- 市场概率可以用平均或中位数，但下注 EV 必须用真实可执行赔率。

### P1：模型权重目前是手工权重，缺少回测校准

位置：

- `worldcup_predictor/models.py` 的 `ModelConfig`
- `worldcup_predictor/predictor.py:54-103`

问题：

Elo、FIFA、主场、体能、旅途、小组形势、历史交锋、国家关系、商业因素等权重目前都是人工设定，没有用历史比赛回测拟合。

风险：

模型解释性较强，但预测概率未校准，不能证明长期正期望。

建议整改：

- 建立 API 原始快照表。
- 保存预测时点赔率。
- 赛后录入 90 分钟赛果。
- 用 Brier Score、Log Loss、ROI、CLV 做回测。
- 回测通过前，报告中继续保持“研究/纸上模拟”口径。

### P2：最高概率比分没有同步应用平局增强

位置：

- `worldcup_predictor/predictor.py:33-48`

问题：

胜平负概率计算时应用了 `draw_boost`，但最高概率比分来自原始比分矩阵，没有同步平局增强。

风险：

比分列表和胜平负概率在强德比/高平局倾向场景下可能轻微不一致。

建议整改：

- 要么让比分矩阵同步应用 draw boost。
- 要么报告中说明比分来自原始进球分布，胜平负经过情景修正。

### P2：历史交锋边际过于简化

位置：

- `worldcup_predictor/auto_predict.py:207-228`

问题：

当前历史交锋只按近 10 场净胜球做衰减加权，没有区分赛事级别、年代、阵容变化、主客场和是否正式比赛。

风险：

对国家队长期历史交锋可能产生噪音。

建议整改：

- 正式比赛权重大于友谊赛。
- 近 3 年权重大于更早比赛。
- 若双方阵容变化大，降低 H2H 权重。

## 三、整改优先级

### 必须先修

1. 今日随机比赛只抽赛前比赛。已完成第一版。
2. 球队评分缺失时强制 WATCH，除非 Elo/FIFA 兜底可用。已完成第一版，Elo/FIFA 兜底待做。
3. 报告和页面明确模拟舱 EV 的概率来源。已完成第一版。

### 下一批修

1. 保存 bookmaker 级别赔率，不再用平均赔率做下注 EV。
2. 增加 API 原始响应快照表。
3. 接入 Elo/FIFA 静态评分底座。

### 后续增强

1. 回测框架。
2. 赛后结算模拟舱。
3. 小组积分和出线形势模型。
4. 伤停、首发、天气和 xG。

## 四、当前可交付口径

当前版本可以作为：

- 本地可运行原型。
- API-Football 自动取数演示。
- 中文报告导出演示。
- 正期望模拟舱框架演示。

当前版本不应表述为：

- 已验证能长期盈利的投注模型。
- 已完成回测的正期望系统。
- 可以直接替代人工风控的赛前决策系统。
