# 世界杯预测交付说明

日期：2026-06-06

## 一、交付内容

- 本地网页程序：`http://127.0.0.1:8765`
- 真实赛事分析入口：页面只针对赛前真实比赛生成分析，初次打开不自动新增预测。
- API-Football 自动获取赛程、双方近 10 场有效赛果、优先级庄家全场赔率、球队统计和历史交锋。
- 石墨灰玻璃工作台界面、橘色信号 Logo，以及“预测工作台 / 数据处理 / 模型审计 / 历史报告”分区。
- 数据处理复核板块：展示双方近期有效比赛、特征结果与线性趋势图。
- 模型输入层：近期比赛已加入时间衰减、对手强度校正和国家队/青年队强度先验，用于降低短样本误导。
- 中文 Excel 报告导出。
- 中文 PDF 报告导出。
- SQLite 本地预测留档。
- 优先级庄家全场盘口结构化快照、赛后结果同步与模型校准验收面板。
- 批量赛事池：支持指定多个 API-Football fixture_id、今日甲级联赛批量建库、批次历史恢复与筛选。
- 每日官方批次：批量结果可设为某日官方复盘批次，同一日期同一范围只保留一个官方批次。
- 赛后复盘：页面可直接查看已结算比赛、待结算比赛、EV 候选真实结果，并导出中文 Excel。
- 模拟舱账本：只有正式纸上观察记录才进入本地账本，赛果同步后自动结算盈亏；当前正式 EV 未开放时资金占用保持为零。
- 实盘准入判定：新增 `/api/live-readiness` 和模型审计页“真实资金闸门”，逐项显示离实盘下注系统还差哪些验证条件。
- 双击启动脚本：`start_worldcup_predictor.command`
- 依赖安装脚本：`setup_worldcup_predictor.command`
- 公网受控分享部署配置：`render.yaml` 与 `docs/online_sharing_deployment.md`
- 最终落地交付报告：`docs/final_delivery_report_2026-06-06.md`
- 实盘下注系统验证与上线计划：`docs/live_betting_validation_plan.md`

## 二、首次使用

1. 双击 `setup_worldcup_predictor.command`。
2. 等待依赖安装完成。
3. 双击 `start_worldcup_predictor.command`。
4. 浏览器会自动打开世界杯预测页面。

如果浏览器没有自动打开，手动访问：

```text
http://127.0.0.1:8765
```

## 三、API Key

API-Football 密钥可填在页面的 `API 密钥` 输入框，也可写入 `.env`：

```text
API_FOOTBALL_KEY=你的_api_key
API_FOOTBALL_RETRIES=3
```

`.env` 只保存在本机，不会上传。

公网分享时，不在访客页面填写 API Key。应在托管平台配置新的服务器侧 `API_FOOTBALL_KEY`，并设置访问口令；详见 `docs/online_sharing_deployment.md`。

## 四、使用流程

### 真实赛事分析

1. 输入主队和客队。
2. 点击“搜索比赛”，选择具体比赛。
3. 也可点击“今日甲级联赛”选择赛程，或点击“随机抽取”直接抽取今日赛前比赛。
4. 点击“生成赛前分析”。
5. 预测完成后导出 Excel 或 PDF 报告。
6. 比赛结束后在“校准与回测准入”区域点击“同步赛果”，累计合格赛前样本，并自动结算模拟舱账本。
7. 切换到“数据处理”，核对近 5–10 场样本、处理过程及线性趋势图；单次赔率快照不会伪装为赔率走势。
8. 切换到“批量赛事池”，可恢复历史批次、设为每日官方批次、查看多场推荐摘要。
9. 切换到“赛后复盘”，按日期查看已结算、待结算和 EV 候选复盘，并导出复盘 Excel。

## 五、产品边界

- 只看 90 分钟赛果。
- 只做纸上模拟舱，不连接真实投注平台。
- 当前版本是“实盘验证系统”，不是已验证盈利的真实下注系统；未通过实盘准入判定前，禁止使用真实资金。
- 页面可展示满足盘口与保守门槛的候选 EV；在 `pfinal` 尚未完成校准回测验证前，API 模式统一降级为待校准复核，不占用模拟资金。
- API 数据缺失时会显示市场缺失或观望。
- 本产品不承诺收益，预测结果用于研究和回测。

## 六、验收建议

1. 打开页面，确认显示真实赛事检索入口，且未自动新增预测运行记录。
2. 使用“搜索比赛”“今日甲级联赛”或“随机抽取”选择一场真实赛前比赛并生成分析。
3. 点击“导出 Excel”，文件可打开且内容为中文。
4. 点击“导出 PDF”，文件可打开且内容为中文。
5. 查看“数据质量”和“模拟舱”，确认低质量数据不会强行下注。
6. 核查报告中的指定庄家、赔率抓取时间和双方近期有效比赛数量。
7. 查看“校准与回测准入”：当前应明确显示样本不足、正式 EV 关闭，并显示隔离的赛后快照数量。
8. 查看“数据处理”中的近期比赛明细和特征表，并确认已收录名称优先显示中文、未收录球队保留 API 英文原名以便识别与后续核定。

## 七、正式模拟信号基线

- 预测数据每队请求最近 10 场，双方各至少 5 场有效的 90 分钟结果。
- 正式赔率来源按优先级选取，默认 `Pinnacle > Bet365 > Betfair > SBO > 10Bet > 1xBet`；首选缺失时按优先级回退，并记录每个市场实际使用庄家。
- 胜平负只接受同一家完整全场 1X2，大小球和让球只接受同一家、同盘口线的全场成对赔率，不跨公司拼盘口。
- 胜平负方向的 `pbase` 基础概率低于 40% 时不形成候选方向。
- `pbase` 与当前实际市场基准去水概率 `qmkt` 相差超过 15 个百分点时只供复核。
- 基础研究 EV 至少 5%、优势至少 8 个百分点、纸上 EV 至少 3% 才可能形成候选方向。
- 大小球和让球当前仅展示逐比分结算的研究 EV 审计；比分分布层未完成专项校准前，paper_EV、p_adj、shrink_k 和正式 EV 均不开放。
- 页面展示融合概率不等同于正式 `pfinal`；只有 `pfinal` 通过时间切分校准与回测验收后，才能启用正式模拟信号。
- 当前本地库已保存 `173` 条预测、`120` 条 API 快照、`2074` 条结构化报价、`20` 场赛果和 `7` 个批量批次；合格校准样本为 `15/100`，仍未达到正式 EV 验收最低要求。
- 输入层校正不等于 `pfinal` 已通过验收；正式 EV 和真实资金仍保持关闭。
- 当前时间切分校准验收仅覆盖胜平负；大小球与让球需另外完成比分分布层验收。
